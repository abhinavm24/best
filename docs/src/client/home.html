Getting Started with Best

Best allows you to write benchmarks in the same way you write unit tests. This allows you to integrate Best into your CI workflow to create a consistent picture of your code's performance over time. Ready to jump in? Check out the Developer Guide to read about how to get started!

import fib from '../fib';

describe('js-execution', () => {
	benchmark('fibonacci 15', () => {
		run(() => {
			return fib(15);
		})
	});

	benchmark('fibonacci 38', () => {
		run(() => {
			return fib(38);
		})
	});
});

Why Use Best?

Reproducible Results

Best is designed to run on dedicated hardware, this means that you are running your benchmarks in the same environment everytime. We encourage you do this so that your results are not effected by an inconsistent environment which can be a false flag for performance changes.

The Best Hub and Agent model works great for implementing this, if you want to learn more take a look at Setting Up a Best Hub.

Expressive Metrics

Best comes packed with ability to measure all the types of metrics you might want to know. You can learn more about the following metrics here.

  • aggregate The total time your benchmark took to run.
  • script The time it took to evaluate your benchmark code.
  • paint If you benchmark involves the DOM, we can measure how much time the browser spent on painting.
  • layout If you benchmark involves the DOM, we can measure how much time the browser spent on layouts.

GitHub Integration

If your team uses GitHub then you can easily create a GitHub App so that you can integrate Best into your Pull Request workflow.

Best comes with a built-in GitHub bot that can create Checks and comment when a pull request is causing significant performance change. If you want to learn more, check out Integrating Best with GitHub.

The Best Frontend

Best comes built-in with a frontend dashboard that allows you to monitor your benchmarks over the time with each commit. This allows you to pinpoint which commits causes improvements or regressions to your performance.

The frontend allows you to view all your projects and benchmarks in one place. If this seems like something you want, you can read through Enabling The Best Frontend.
